flag按照个人理解就是一个判断某个if语句或者是循环语句是否发生的标志，一般判断if语句是否发生变化，可以用
flag=0，和flag=1解决，若在一般循环语句中，判断是否循环发生，也是同样的道理。但如要判断是否为第一次循环，
或是否其中的某一次循环，flag则可以表示为flag++。

关于输入的循环

while(~scanf(“%d”,&a)等价于while(scanf(“%d”,&a)!=EOF)等价于while(scanf(“%d”,&a)==1),约等于while(scanf(“%d”,&a),a)但值得注意的是，while(scanf(“%d”,&a),a)其返回值是0；即输入0立即结束，输入面较其他几种较低，而while(scanf(“%d”,&a)==1)其返回值为1，但因为前面定义输入是整数型，若输入字符则会结束，所以目前较为合适的应用应为前两种。个人推荐第一种，字少。。。
ps:循环输入字符数组时，用while（gets（X）！=NULL）；

做题最简单的一些坑

判断或者循环里的判断，“=”与“==”一个符号差距就是个人心理崩不崩溃的问题，一般输出表现为，单个数值的溢出，多组输出
定义char，char被空格吃了，这种在输入里的问题最难发现，但是也是最好解决的，比如scanf(“%c%c”,&a,&b)只要在输入里不偷懒，直接在两个%c之间加空格就行了。

近日学完了函数，关于函数部分问题；
首先，关于自己定义的函数，如果是有参函数，且参数是数组，不能直接定义为，int/void XX（int a[XX]）或者int/void XX（int a），而应该定义为int/void XX（int a[]）;其其次，在主函数或者是其他函数中调用此函数（有参）时，并不需要声明，可以，直接XX（参数），而如果在之前画蛇添足的加一个int XX；则会报错；对于无参函数，直接引用即可。
关于函数的递归，即分级处理，把原本用一个代码可以处理的问题（当然，这串代码相当复杂，直接进行逻辑思考，可能思维会混乱），分为许多小部分（每个部分都有自己的作用）。总的来说，就是把一个复杂的大问题，分解为许多个细小的小问题来解决。
ps：个人推荐把自己定义的函数放在主函数前。

关于一些其他方面的进度
acm竞赛入门基础电子版进度依然在函数，因为此节得知识有点难以理解，而且题型也开始变的多解（个人喜欢试着用两种方法解解看，但多数时候只会用一种），这本电子书里很多细节时老师没说，和书上没有的，可以作为笔记。

关于指针的一些问题
指针*p，说简单点就是门牌号，说复杂一点就是数据的地址量（ps：具体的地址可以把数值地址赋给输入量X后，用%p输出十六位地址）；用指针引用一维数组时，可以直接p=a；此时p即获得a[0]的地址量；但对于二维数组，p=a不可使用（即使二维数组的地址量也是连续的）；那么对于二维数组，假设此时定义为a[3][3]，那么定义指针时可以定义为（p）[3]，此时再运用p=a，p的初始地址量即为a[0][0]，当然，如果不擅长这种定义，那么只要能把a数组的首地址赋予给p即可，即p=&a[0][0]。

春天是鲜花的季节，水仙花就是其中最迷人的代表，数学上有个水仙花数，他是这样定义的：
“水仙花数”是指一个三位数，它的各位数字的立方和等于其本身，比如：153=1^3+5^3+3^3。
现在要求输出所有在m和n范围内的水仙花数。

Input
输入数据有多组，每组占一行，包括两个整数m和n（100<=m<=n<=999）。

Output
对于每个测试实例，要求输出所有在给定范围内的水仙花数，就是说，输出的水仙花数必须大于等于m,并且小于等于n，如果有多个，则要求从小到大排列在一行内输出，之间用一个空格隔开;
如果给定的范围内不存在水仙花数，则输出no;
每个测试实例的输出占一行。

Sample Input
100 120
300 380

Sample Output
no
370 371
